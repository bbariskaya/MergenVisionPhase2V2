6. GPU HOT-PATH CONTRACT
======================================================================

Production-like bulk compute path:

encoded JPEG bytes
→ GPU decode
→ CUDA preprocess
→ batched TensorRT RetinaFace
→ GPU decode/NMS/landmarks
→ GPU quality primitives
→ CUDA five-point alignment
→ batched TensorRT GlintR100
→ GPU L2 normalization
→ compact extraction results

CPU boundary’de izin verilen:

- Input encoded JPEG bytes
- File/manifest metadata
- Original image dimensions
- bbox
- landmarks
- detector score
- quality primitives
- 512-D normalized embedding
- compact encoded aligned crop
- status/rejection reason
- timings

Yasak:

- Production batch path’te PIL/OpenCV full-frame decode
- Full decoded image D2H
- Detector input/output tensor’larını CPU’ya taşıma
- CPU NMS/postprocess
- CPU model inference fallback
- Fotoğraf başına ayrı Python→GPU inference çağrısı
- Fotoğraf başına zorunlu CUDA synchronize
- Sessiz sample truncation

“Hiç GPU’dan inmeden” iddiasını teknik olarak dürüst kullan:

- Encoded dosya byte’ları başlangıçta CPU’dadır.
- Persistence için compact sonuçların CPU boundary’ye çıkması gerekir.
- İddia: decoded full image ve intermediate inference tensor’ları GPU hot path’ten çıkmaz.
