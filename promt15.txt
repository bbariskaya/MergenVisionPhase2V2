14. GERÇEK DATASET BATCHING
======================================================================

Mevcut subject-by-subject GPU call kaldırılmalıdır.

Yanlış:

  for subject:
      extract_batch(subject.photos)

Doğru:

- Manifest streaming okunur.
- PhotoTask’lar farklı subject’lerden batch’e girebilir.
- Her task stable association taşır:

  source_namespace
  subject_key_hmac
  face_id
  sample_id
  content_sha256
  source ordinal/path handle

- Worker bir shard içindeki farklı subject fotoğraflarını microbatch yapar.
- Result source key üzerinden doğru subject/sample’a geri bağlanır.
- Partial final batch işlenir.
- Yüz sessizce düşürülmez.

Dataset’in tamamı RAM’e yüklenmez.

Bounded queues:

  manifest/read queue
  GPU inference queue
  persistence queue
  result queue

Her queue explicit maxsize taşır. Persistence yavaşsa GPU producer’a backpressure
uygulanır.

Input formatı:

- read-only filesystem bind mount
- versioned manifest/JSONL
- dataset API’ye base64 olarak gönderilmez
- dataset MinIO’ya topluca kopyalanmak zorunda değildir
- source dosyalar değiştirilmez
- source fotoğraflar accepted artifact olarak saklanmaz

======================================================================
15. GERÇEK MULTI-GPU
======================================================================

`--gpu-devices 0,1,2` gerçek üç worker oluşturmalıdır.

Kurallar:

- Bir process/container tam olarak bir fiziksel GPU görür.
- Container içindeki CUDA device index normalde 0’dır.
- Host GPU UUID/index mapping manifestte kaydedilir.
- Her worker:
  - kendi CUDA context’i
  - kendi warmed pipeline’ı
  - kendi engine/context/buffer arena’sı
  taşır.
- Python built-in `hash()` kullanılmaz.
- Sharding:

  stable_hmac(subject_key) mod worker_count

- Aynı subject’in bütün fotoğrafları aynı worker’a gider.
- photo_id ile sharding yapılmaz.
- Worker crash/retry aynı subject’i başka ID ile yeniden yaratmaz.
- Worker exit sonrası GPU memory baseline’a dönmelidir.

Single GPU mode da çalışmalıdır.

Conflict handling:

- Inactive/failed/conflicting sample sessizce reactivate edilmez.
- UPDATE previous-state predicate kullanır.
- rowcount kontrol edilir.
- Ownership mismatch hard failure olur.
