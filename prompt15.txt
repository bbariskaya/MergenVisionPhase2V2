1. HOT-PATH SINIRI
============================================================

Zorunlu akış:

encoded JPEG batch
  → nvImageCodec GPU decode
  → device RGB images
  → GPU RetinaFace preprocess
  → TensorRT RetinaFace
  → CUDA decode/NMS/landmarks
  → CUDA five-point alignment
  → device-side uint8 RGB 112×112 aligned crop batch
  → TensorRT GlintR100 preprocess/inference
  → CUDA L2 normalization
  → nvImageCodec/nvJPEG GPU-accelerated JPEG encode
  → compressed JPEG bitstreams
  → bounded batch persistence
  → PostgreSQL / MinIO / Qdrant

Full decoded source frame CPU’ya dönmeyecek.

CPU boundary’de yalnız şunlar olabilir:

- bbox
- landmarks
- score/rejection reason
- 512-D normalized embedding
- sıkıştırılmış aligned JPEG bytes
- IDs ve teknik metadata

alınması normal ve izinlidir. Raw 112×112 RGB crop veya full frame’i Python
PIL/OpenCV’ye taşımak yasaktır.

============================================================
3. GPU JPEG ENCODER TASARIMI
============================================================

Öncelikli çözüm:

  nvidia.nvimgcodec.Encoder

Girdi:

- GPU device memory
- contiguous HWC
- uint8
- RGB
- shape B×112×112×3
- `__cuda_array_interface__` veya DLPack üzerinden
- mevcut CUDA stream ownership korunarak

Encoder:

- device_id worker GPU
- batch/list encode
- codec=`jpeg`
- quality=95
- cuda_stream=current pipeline stream

Backend explicit sınırlandırılmalı:

1. Runtime destekliyorsa `nvjpeg_hw_encoder`.
2. Değilse `nvjpeg_cuda_encoder`.
3. `opencv_jpeg_encoder` kesinlikle yasak.
4. CPU/PIL fallback kesinlikle yasak.
5. Requested NVIDIA backend yoksa startup:
   `BLOCKED_GPU_JPEG_ENCODER_UNAVAILABLE`
   dönmeli.

Backend seçimini varsayma. Runtime smoke ile gerçekten kullanılan encoder
backend’ini ve version’ı kaydet.

Eğer Python nvImageCodec wrapper mevcut DeviceTensor’ı güvenli biçimde kabul
etmiyorsa:

- PIL’e dönme.
- Raw crop’u NumPy’ya indirme.
- `_mv_phase1_bulk_native` içine native `NvJpegBatchEncoder` binding ekle.
- Binding device pointer + pitch + batch + stream kabul etsin.
- nvJPEG encoder state/params reusable olsun.
- Her crop için handle/state create-destroy yapma.
- Pinned host bitstream buffer pool kullan.
- Encode tamamlandıktan sonra compressed byte’ları batch halinde Python’a çıkar.

============================================================
4. CROP LAYOUT
============================================================

Recognizer input ve artifact crop aynı şey değildir.

Artifact encoder için gerekli crop:

  uint8 RGB HWC 112×112

GlintR100 input:

  FP32 NCHW 112×112
  model profile normalization contract’ına uygun

Alignment aşamasında mümkünse bir canonical uint8 RGB crop batch üret.

Ardından GPU kernel:

  uint8 HWC aligned crop
    → GlintR100 normalized FP32 NCHW

dönüşümünü yapsın.

Böylece:

- artifact JPEG aynı canonical alignment’tan gelir
- recognizer aynı pixel source’u kullanır
- ikinci alignment yapılmaz
- CPU pixel conversion yapılmaz
- crop/embedding association kaybolmaz

============================================================
5. BATCH ENCODE / IO
============================================================

Encode photo-by-photo Python loop şeklinde blocking yapılmayacak.

İstenen aşamalar:

GPU worker:

- detector batch
- selected crops
- recognizer chunks
- JPEG encode batch
- compact result batch

Persistence worker:

- compressed JPEG byte batches
- PostgreSQL batched pending reserve
- MinIO bounded concurrent put
- Qdrant chunked upsert wait=True
- PostgreSQL batched active finalize

Queue’lar bounded olacak. Persistence yavaşsa GPU tarafında sınırsız RAM
birikmeyecek.

MinIO SDK tek gerçek “batch put” sağlamıyorsa bounded async concurrency kullan:

- explicit concurrency limit
- byte/count based chunk limit
- per-object idempotent key
- retry/backoff
- previous successful object’i silmeme
